ADMIN CLIENT PROFILE — BONUS SESSIONS SYSTEM + CLIENT DISPLAY

Goal:

Admin must be able to enter ANY client profile and manually add bonus sessions directly.

No new order.
No package purchase.
No checkout.
No payment flow.
Pure admin adjustment.

LOCATION:

Control Panel →
Client Profile →
Package / Sessions Section →
Quick Actions

Create premium Tron Legacy style action:

✨ Bonus Sessions

WHEN CLICKED:

Open premium Tron-style modal.

Fields:

1. Bonus Sessions Count
Input: 1–100

2. Reason
Dropdown:
- Promotion Bonus
- Renewal Bonus
- Compensation
- Loyalty Reward
- VIP Gift
- Manual Adjustment
- Other

If “Other”:
Show custom text field.

3. Expiry Extension optional
Options:
- No extension
- Extend by days
- Set custom expiry date

4. Internal Admin Note optional

BEHAVIOR:

Added bonus sessions must:

- Increase remaining sessions
- Increase total available sessions
- Keep booking system working normally
- Be usable for bookings

Do NOT:

- Create order
- Create invoice
- Trigger payment
- Change payment status
- Modify purchased package price
- Affect paid sessions count

CLIENT DASHBOARD DISPLAY:

Client must clearly see the difference between purchased package sessions and bonus sessions.

Show in Package / Sessions card:

- Package Sessions: original paid sessions
- Bonus Sessions: added bonus sessions
- Total Available Sessions: package sessions + bonus sessions
- Used Sessions
- Remaining Sessions
- Expiry Date

Example:

Package Sessions: 20
Bonus Sessions: +3
Total Available: 23
Used: 5
Remaining: 18

Important:

Do NOT hide bonus sessions inside total only.

Client should clearly understand:

“I bought 20 sessions and received 3 bonus sessions.”
