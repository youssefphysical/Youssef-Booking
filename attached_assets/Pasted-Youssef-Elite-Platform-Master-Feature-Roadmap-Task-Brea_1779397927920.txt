Youssef Elite Platform — Master Feature Roadmap & Task Breakdown

Important:
Do NOT implement everything at once.

First, read this full roadmap carefully.
Then create a clear task checklist grouped by phases.
Implement one phase at a time only.
After each phase, test all existing flows before moving to the next phase.

Main philosophy:

- Client experience must stay simple, premium, and effortless.
- Admin side can be powerful and detailed.
- Do not overload the client UI.
- No dead-end screens.
- No confusing flows.
- One clear primary action per screen.
- If the user needs to think too much, the UX failed.

BRAND:
Use consistent brand name:
Youssef Elite

Keep Fitness Zone only when referring to the actual gym/location.

PHASE 1 — Core Booking, Wizard, Package & Access Logic

1. Homepage CTA

- Main CTA must go to "/wizard", not "/book".
- New users must choose training type/location first.
- "/book" must never flash or show skeletons before wizard completion.

2. Wizard flow
   User types:

- Brand new user
- Existing Fitness Zone PT client
- Home / building / hotel training
- Another gym

3. Brand new user flow

- User selects “I’m brand new”.
- Continue sends user to free trial booking.
- Free trial does not require active package approval.
- Session type should be "free_trial".
- Prevent duplicate free trial abuse using phone/email/device/account checks.

4. Existing Fitness Zone PT client flow

- User selects package only:
  - 10 sessions
  - 20 sessions
  - 25 sessions
  - Duo 30
  - Not sure
- No receipt upload.
- No purchase receipt message.
- Submit button: “Submit for Approval” or “Request Package Activation”.
- After submit, create pending package activation request.

5. Pending approval logic

- User cannot book until admin approves.
- If user opens "/book" manually while pending, redirect to dashboard/profile pending status.
- No booking calendar.
- No booking skeleton.
- No confusing loading.

6. After submission UX

- Show loading: “Sending your request...”
- Redirect to dashboard/profile.
- Show success notification:
  “Package activation request received.”
- Show package status card:
  Title: Package Activation In Review
  Status: Pending Review
  Package: selected package
  Submitted: date/time
  Message:
  “The Youssef Elite team will review your package and unlock your booking access once activation is complete.”

7. Approved package logic

- Admin approval activates the selected package.
- Booking access unlocks automatically.
- Sessions count is assigned correctly.
- Pending state disappears.
- Package becomes Active.
- User sees “Book Session”.

8. Booking protection

- Prevent double booking.
- Prevent duplicate fast-click booking.
- Prevent booking less than 6 hours before start time.
- Enforce trainer availability.
- Enforce package expiry.
- Enforce active package/session credits.
- Enforce cancellation policy.
- Emergency cancel: 1 per month.
- Auto-lock expired or fully used packages.

9. Waitlist

- If a slot/day is full, allow Join Waitlist.
- If a slot opens, notify the first suitable waitlisted client.

10. Add to Calendar

- After confirmed booking, allow Add to Google Calendar / Apple Calendar.
