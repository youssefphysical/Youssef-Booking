Goal:

Steps:

1. Final Safety Verification
Run:
- npm run check
- npm run build

Verify:
- No TypeScript errors
- Build succeeds
- No route errors
- No admin protection regression
- No authentication regression
- No booking regression
- No package regression
- No dashboard regression

2. Verify Production Configuration
Check:
- vercel.json exists and is valid
- Build command is correct
- Output/start settings are correct for Vercel

- SPA fallback does not intercept API routes
- Admin APIs return 401 JSON when unauthenticated
- Public routes return 200 where expected
- ensureSchema runs safely and additively
- No secrets are hardcoded
- No stack traces exposed

3. Verify Neon PostgreSQL Readiness
Check:
- App uses DATABASE_URL from environment variables
- SSL-compatible connection if required
- Migrations / ensureSchema are safe
- No destructive migrations
- No data reset
- No hardcoded local database URL

4. Document Required Vercel Environment Variables
Create or update a deployment notes file if needed and list:

DATABASE_URL=
JWT_SECRET=
SESSION_SECRET=
RESEND_API_KEY=
PUBLIC_APP_URL=
CRON_SECRET=
OPENAI_API_KEY=
NODE_ENV=production

Also mention:
- JWT_SECRET and SESSION_SECRET must be strong random values.
- PUBLIC_APP_URL must match the final Vercel/domain URL.
- DATABASE_URL must be the Neon production connection string.
- RESEND_API_KEY is required for emails.
- CRON_SECRET is required for protected cron jobs if used.
- OPENAI_API_KEY only if OCR/AI features depend on it.
