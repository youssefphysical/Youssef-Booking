Build a complete client onboarding + package selection + approval workflow system.

IMPORTANT:
- Do NOT break booking, login, admin panel, or existing data.
- This is an extension, not a rewrite.

========================
1. Client Signup Flow
========================
During account creation:

Client must select:
- Full name
- Email
- Phone number
- Goal:
  - Fat loss
  - Muscle gain
  - Recomposition
  - General fitness
- Package (from admin-created packages)

Show packages clearly:
- Package name
- Sessions (e.g. 10+1, 20+3)
- Price
- Expiration duration

After signup:
- Create client profile
- Status = "Pending Approval"
- Do NOT allow full booking yet (restricted access until approval)

========================
2. Admin Notification
========================
Immediately after signup:

Send notification to admin (in dashboard and optionally email/log):
- Client name
- Email
- Phone
- Selected goal
- Selected package

Add Admin “New Client Request” panel:
- List all pending clients
- Show full info
- Buttons:
  - Approve
  - Reject

========================
3. Admin Approval Logic
========================
When admin clicks APPROVE:

- Activate client profile
- Assign selected package
- Set:
  - Total sessions
  - Bonus sessions
  - Remaining sessions
  - Start date
  - Expiration date (auto based on package)

When admin clicks REJECT:
- Keep profile but mark as rejected
- Block booking

========================
4. Booking Integration
========================
Client can:
- View booking system immediately after signup
- BUT:
  - Cannot confirm booking unless approved

After approval:
- Booking becomes fully active
- Each booking deducts 1 session

========================
5. Session History System
========================
Track EVERYTHING:

For each session:
- Date
- Time
- Status:
  - booked
  - attended
  - cancelled
  - no-show
  - rescheduled
- Which package it belongs to
- Admin notes

For manual changes:
- Add log:
  - +1 session
  - -1 session
  - reason
  - date

========================
6. Package Control (Admin)
========================
Admin can:
- Increase sessions
- Decrease sessions
- Add bonus sessions
- Change expiration
- Freeze/unfreeze
- Change package

All changes must be logged.

========================
7. Full Date Access
========================
Admin must see:
- ALL sessions (past + future)
- Filter by:
  - client
  - date
  - status

========================
8. Session Content Control
========================
Admin can:
- Add session notes
- Add training focus per session:
  - chest
  - back
  - legs
  - etc.
- Add goal per session

Client can VIEW but not edit.

========================
9. Dashboard Search (CRITICAL)
========================
Admin dashboard must include powerful search:

Search by:
- Name
- Phone number
- Email

Must be:
- Fast
- Real-time
- Works on mobile

========================
10. Admin Dashboard UX
========================
Improve usability:

- Add tabs:
  - Clients
  - Pending Requests
  - Sessions
  - Packages
- Make layout clean and easy
- Reduce clutter
- Make buttons large and clear
- Mobile friendly
- No overflow or broken layout

========================
11. Client View
========================
Client sees:
- Selected package
- Total sessions
- Remaining sessions
- Expiration date
- Booking access (if approved)

Client CANNOT:
- Change package
- Change sessions
- Approve themselves

========================
12. Safety
========================
- Do NOT delete existing clients
- Existing users without package = show "No package assigned"
- Keep all old booking data
- No breaking changes

========================

Test:

- Booking before/after approval
- Session deduction
- Search
- Mobile UI
