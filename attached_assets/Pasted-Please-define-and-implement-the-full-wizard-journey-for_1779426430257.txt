Please define and implement the full wizard journey for ALL training-location flows, not only Fitness Zone.

The user must always know:

- What happens after clicking the button
- Where they will be redirected
- What they will see on the next page
- What is locked/unlocked
- What the next action is

Global rule:
After any successful wizard action, the user must never stay stuck on the same wizard page.

All flows must have:

- Loading state
- Disabled button during submit
- Success toast
- Correct redirect
- Dashboard/profile status card if needed
- Notification bell event if needed
- Clear error toast if something fails

FLOW 1 — Existing Fitness Zone PT Client

User selects:
“I’m already a Fitness Zone PT client”

User selects package:
10 / 20 / 25 / Duo 30 / Not sure

User clicks:
“Submit for approval”

After click:

- Button: “Sending request...”
- Disable duplicate submit

On success:

- Toast: “Package activation request received.”
- Redirect to "/dashboard"

Dashboard shows:
Title:
Package Activation In Review

Status:
Pending Review

Details:

- Selected package
- Submitted date/time

Message:
“The Youssef Elite team will review your package and unlock your booking access once activation is complete.”

Timeline:

1. Request received
2. Under review
3. Booking access unlocked

Booking:

- Locked until admin approval
- If user opens "/book", redirect back to dashboard with locked message

Notification:
“Package activation request received”

After admin approval:

- Package becomes Active
- Booking unlocks
- Sessions count assigned
- Dashboard shows Book Session button

FLOW 2 — Brand New User

“Continue”

Notification:

ERROR HANDLING FOR ALL FLOWS

If any save/submit fails:

- Stay on wizard
- Show visible toast:
  “We couldn’t complete your request. Please try again.”
- Button returns to normal
- Do not silently fail

If route guard blocks booking:

- Redirect dashboard
- Show clear locked/pending status card

If user refreshes:

- Preserve selected state if possible
- Do not lose draft location details
