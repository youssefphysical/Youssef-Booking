GLOBAL PREMIUM INTERACTION / TAP-FEEDBACK REFINEMENT PASS

The current UI has a major premium-quality issue across the entire platform:

PROBLEM:
Whenever the user taps/clicks selectable UI elements (chips, cards, time slots, dates, selectors, buttons, toggles, booking options, package cards, session focus chips, calendar cells, dropdown options, etc.), a default gray/rectangular flash appears for a fraction of a second before the actual selected state appears.

This creates:
- cheap browser-like behavior
- inconsistent interaction language
- non-premium UX feeling
- “laggy” visual perception
- broken AMOLED luxury aesthetic

The platform must feel like a polished high-end premium mobile application, not default browser controls.

CRITICAL REQUIREMENT:
REMOVE ALL DEFAULT TAP/CURSOR/ACTIVE/FOCUS FLASHES GLOBALLY.

This includes:
- browser tap highlight
- default active background
- default radix/shadcn active flashes
- default focus flashes
- default mobile web tap rectangles
- default selection gray overlays
- any white/gray flicker before selected state

Apply this globally across the ENTIRE website.

====================================================
ANIMATION SPEC
====================================================

Use:
- transition-all
- ultra-smooth easing
- 150–220ms timing
- subtle scale only

Pressed state:
- scale: 0.97–0.985
- opacity shift: very subtle
- glow intensity slightly increases
- background softly deepens

Selected state:
- smooth animated transition
- no instant snapping
- no delayed checkmark pop-in
- selection indicator fades/slides naturally

====================================================
COLOR-AWARE FEEDBACK
====================================================

IMPORTANT:
The press/tap effect MUST inherit the color identity of the component itself.

Examples:

CYAN components:
- cyan glow
- cyan active tint
- cyan ripple/overlay

AMBER/GOLD selected dates:
- warm amber pulse
- soft gold glow
- luxury golden pressed state

RED warning cards:
- subtle red active tint

GREEN success buttons:
- green pressed feedback

NEVER use generic gray pressed overlays.
