Refactor ONLY the Admin Global Search into a professional Client Search Center.

IMPORTANT:
Everything else is working. Do NOT touch bookings logic, clients logic, packages logic, auth, payments, database schema, Neon, Vercel, GitHub, Media, Logo Controls, uploads, colors, theme, or unrelated pages.

Goal:
The current search is too generic. It should NOT be a command palette for the whole admin. Convert it into a focused Client Search Center only.

Main concept:
Search = find/manage clients.
Plus button = create actions.
Navigation = page movement.

Remove from search:

- Dashboard
- Bookings page shortcut
- Packages page shortcut
- Media shortcut
- Settings shortcut
- New client
- New booking
- New package template
- Any general command/action not directly related to a matched client

Search should only search clients by:

- client name
- phone number
- email
- package name
- client status
- VIP/active/healthy/package labels if already available

Client result card requirements:
Each result should show:

- Client full name
- Email
- Phone
- Current package name if available
- Sessions remaining if available
- Client status/package badge if available
- Optional last session date if already available without heavy extra query

Client result actions:
Main row click/tap opens client profile.
Add compact actions only if already supported safely:

- Open Profile
- Book Session
- Add Package
- Add Bonus Sessions

Do not create new backend logic unless needed. Prefer existing routes and existing client data.

Empty state:
Before typing, show a clean professional empty state:
Title: Search Clients
Text: Search by name, phone, email, or package.
Do not show dashboard/navigation/create actions here.

No results state:
Title: No clients found
Text: Try another name, phone, email, or package.
Optional: show “Create client” only if it uses the existing safe create-client flow.

Ranking:

- Exact name match first
- Prefix name match second
- Phone/email exact or contains match high priority
- Package match below name/phone/email
- Active/VIP/current clients before inactive clients if available
- Do not show unrelated clutter after typing a specific client name

- No horizontal overflow.
- Search input placeholder on mobile:
  “Search clients…”

  “Search clients by name, phone, email, or package…”

- Footer helper text should be minimal or hidden on mobile.

- Cards should be compact, readable, and touch-friendly.

Desktop/tablet:

- Keep it clean and centered.
- Max width should be controlled and not too wide.
- Keyboard navigation can remain if already implemented.
- Enter opens highlighted client.
- Esc closes.
- X clears query.
