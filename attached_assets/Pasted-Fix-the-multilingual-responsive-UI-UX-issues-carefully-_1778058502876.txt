Fix the multilingual responsive UI/UX issues carefully without breaking anything.

Main goal:
Make the website fully consistent across all languages, mobile and desktop, especially the hero section, spacing, RTL Arabic behavior, and typewriter animation.

VERY IMPORTANT:
- Do not break existing booking logic.
- Do not break login/sign-in.
- Do not break admin panel.
- Do not break packages, clients, language switcher, hero slider, or WhatsApp buttons.
- Keep all existing data safe.

1. Multilingual Hero Layout Fix
- Fix the hero section spacing for all languages.
- Add consistent spacing between:
  - badge/label
  - headline
  - subtitle/description
  - CTA buttons
  - slider dots
- The text must never stick to the blue badge.
- Add safe vertical spacing: badge margin-bottom 14px–18px.
- Add safe spacing between headline and subtitle: 16px–20px.
- Add safe spacing between subtitle and buttons: 22px–28px.
- Prevent overlap between text and buttons.
- Prevent any cut-off text.

2. Responsive Typography
- Make headline font size responsive using clamp().
- Example: font-size: clamp(38px, 8vw, 72px) on mobile/hero where appropriate.
- Use proper line-height: 1.12–1.25 for headlines.
- Subtitle line-height: 1.45–1.65.
- Long languages like Spanish, German, Russian, Turkish, Arabic must wrap cleanly.
- Use max-width for hero text so it does not stretch badly.
- Use word-break: normal; overflow-wrap: anywhere only when needed.
- No text should overflow outside the screen.

3. RTL Arabic Support
- When Arabic is selected:
  - Set dir="rtl" correctly on the page/root container.
  - Align hero text to the right.
  - Flip relevant flex directions only where needed.
  - Keep buttons clean and readable.
  - Keep icons positioned naturally.
- Arabic layout must feel native RTL, not just translated text.

4. Typewriter Animation Direction
- Keep the typewriter/dynamic typing animation for all languages.
- For LTR languages:
  - Text should type from left to right.
  - Cursor appears on the right side.
- For Arabic RTL:
  - Text should type from right to left.
  - Cursor appears on the left side.
- The animation must detect language direction dynamically.
- Do not remove the animation unless needed to prevent a bug.
- If reduced motion is enabled, show full text without animation.

5. Language Switching Stability
- When changing languages:
  - No layout jump that breaks the hero section.
  - No overlap.
  - No broken spacing.
  - No clipped text.
  - No horizontal scrolling.
- Ensure the hero section adapts smoothly to each translated text length.
