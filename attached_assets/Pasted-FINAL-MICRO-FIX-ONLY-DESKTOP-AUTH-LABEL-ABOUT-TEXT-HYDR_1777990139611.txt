Problem:
On desktop hard refresh, the header initially shows Sign In correctly, then after auth hydration it changes to Sign Out even when the user is not truly logged in.

Investigate:
- Desktop header auth state only.
- /api/user or auth/me response handling.
- Cookie/session detection.
- Any logic treating loading/unknown/error as authenticated.
- Any static Sign Out button rendered on desktop.
- Any condition like:
  loading → Sign Out
  authChecked → Sign Out
  user object exists but invalid/empty → Sign Out

Correct behavior:
- Logged out / guest / 401 / no valid user → show Sign In only.
- Logged in valid user → show Sign Out/Profile only.
- Never show Sign Out unless there is a confirmed authenticated user with valid id/email.
- Never switch from Sign In to Sign Out after refresh unless user is actually authenticated.

Implementation:
Create a strict desktop auth check:

  Boolean(user && (user.id || user.email));

If API returns 401:
- set user = null
- set authenticated = false
- show Sign In

If API fails:
- treat as guest
- show Sign In
